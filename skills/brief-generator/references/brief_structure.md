# Project Brief Structure Guide

A good project brief should clearly explain what needs to be done and what is expected.

Preferred order:
1. Project Overview
2. Goals
3. Scope
4. Deliverables
5. Timeline
6. Assumptions
7. Risks or Constraints

Rules:
- Keep the content structured and easy to scan
- Use bullet points where useful
- Do not add missing information as facts
- If something is unclear, write "Not specified"
- Keep the tone professional and practical
